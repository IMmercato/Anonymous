# Anonymous
### A social network that actually doesn't know who you are.

---

## What is this?

Anonymous is an Android app I built because I was frustrated with how every social platform wants your phone number, email, location, and basically your entire life before you can send a message. This app lets you chat, share media, and join communities without any of that. No account. No email. No phone number. Nothing.

Your identity on the network is just a cryptographic address generated by I2P — not by me, not by any server, but by the math itself. I can't see who you are even if I wanted to.

The whole thing runs over [I2P (Invisible Internet Project)](https://geti2p.net/), a darknet that routes your traffic through multiple encrypted tunnels. There is no clearnet server behind this. No API sitting on some cloud VM logging requests. The app *is* the network.

---

## How I built it

I started with the idea of embedding an I2P router directly inside an Android app, which turned out to be way harder than I expected. I used [i2pd](https://github.com/PurpleI2P/i2pd) (the C++ I2P implementation) compiled as a native library via the [i2pd-android](https://github.com/PurpleI2P/i2pd-android) JNI bridge.

The UI is built entirely in **Kotlin with Jetpack Compose**. For messaging I use the SAM (Simple Anonymous Messaging) bridge that i2pd exposes locally on port 7656 — it's basically a local socket API that lets me create I2P sessions and stream data through tunnels without touching raw I2P internals.

For encryption, messages are end-to-end encrypted using **ECDH + AES-256-GCM**. Each message uses an ephemeral key exchange so there's no long-term key to compromise. Community group chats use AES-256-GCM with a shared group key embedded in the invite link.

Media (images, GIFs) is chunked into 16KB pieces, sent chunk by chunk over the I2P stream, reassembled on the other end, and verified with SHA-256. Before any image leaves your device, all metadata (EXIF, GPS, device info) is stripped.

Identity is stored locally using Android's EncryptedSharedPreferences and the hardware-backed AndroidKeyStore where available.

## How to run it

> **Android only. This app does not exist on iOS and there are no plans for an iOS version(yet).**

### Requirements
- Android Studio (latest stable) with Kotlin support
- A physical Android device or emulator (API 26+)
- The i2pd native library — already included via the i2pd-android dependency

### Steps

1. Clone the repo:
   ```bash  
git clone https://github.com/IMmercato/Anonymous.git  cd Anonymous ```
2. Open the project in Android Studio.

3. Let Gradle sync and download dependencies.

4. Connect your Android device (or start an emulator) and hit **Run**.

5. On first launch, the app will start the embedded I2P router and wait for it to connect to the network. **This can take 2–5 minutes** on the very first run because i2pd needs to download the network database and build tunnels. Subsequent launches are much faster.

6. Once connected, tap **"Be Anonymous!"** to generate your identity. A QR code is created — this is your address. Share it with friends so they can add you as a contact.

### Notes
- Keep the app open during the first launch — closing it before i2pd finishes reseeding will require it to start over.
- Battery optimization exemption is requested on startup. Grant it if you want the I2P service to stay alive in the background.
- The app needs an active internet connection to reach the I2P network, but your traffic is anonymized through multiple hops before leaving your device.

### Contribution Guidelines
- **Pull Requests:** All contributions must follow the project's coding and security standards. Any networking change must preserve I2P exclusivity — no clearnet fallback may be introduced.
- **Issues:** Open an issue on the GitHub repository for bugs or suggestions.
- **Security Principle:** Any contribution that introduces data collection, clearnet communication, or logging of user activity will be rejected outright.

## Future Vision

Anonymous is not just about creating a secure social network — it's about reimagining how we interact online in a world where privacy is a fundamental right, not a premium feature.

- **Enhanced I2P Integration:** Deeper use of I2P's native streaming and datagram APIs for lower-latency messaging
- **Community Moderation Tools:** Decentralized, anonymous moderation mechanisms that maintain a safe environment without any identity exposure
- **Cross-Platform I2P Clients:** Web interface accessible exclusively via an `.i2p` address, requiring an I2P-enabled browser or proxy
- **Open Protocol (not Open API):** A documented, open communication protocol that developers can implement independently — without any central API dependency

---

## Conclusion

Anonymous stands at the intersection of privacy, innovation, and secure communication. By building entirely on I2P and i2pd — and collecting zero data by architecture — we ensure that anonymity is not a feature that can be switched off, overridden, or compromised. There is no server to seize, no database to breach, no API to surveil.

In this platform, the principle of anonymity isn't just a feature — it's the foundation upon which every function is built.

**Your anonymity is your strength. And here, it is structurally guaranteed.**
